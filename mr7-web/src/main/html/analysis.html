Vodyanoi Analytical Remarks
Vodyanoi is technology used to build cognition network. Based on technologies which are associated with the Semantic Web it aims to make thoughts sharing and exchange as well as cooperation of humans simple and efficient.

Please note that this page is just an informal high-level description of Vodyanoi - distributed system to be built on top of the mind maps created with MindRaider. To become familiar with Vodyanoi you should start with e-mentality blog.

MindRaider Repository

Data maintaned by MindRaider are stored to the two types of files:

  • Resource (RF)
  • RDF model (RMF)

Resource file is created for every resource maintained by MindRaider e.g. folder, notebook, concept, etc. RF is formed by metadata and data part. Typically it holds information like label, annotation, creation and modification timestamps, author and classification of the resource.

RDF model files are used to organize resource files. RMFs in fact form RDF/OWL metadata layer on top of RFs.

If you are familiar with (T)Wiki then there exist the following analogy on the resource type level

  • Web ~ Folder
  • Topic ~ Notebook
  • WikiWord ~ either Notebook or Concept Label

Publishing Notebooks to Vodyanoi

In Vodyanoi user will be allowed to mark arbitrary Notebook as public and publish it to the P2P network of Vodyanoi nodes.

In fact it means that part of the personal mind map will be transformed and published to the server. There are the following requirements on Vodyanoi's Notebook representation:

  • MindRaider 2 Vodyanoi representation transformation must be easy. It must preserve all the information except references to local data stored on the user's computer.
  • It must be possible to transform it back from Vodyanoi 2 MindRaider representation while focusing on the concepts annotation (see below).
  • Representation must be pre-processed in order to simplify semanticaly enriched XHTML generation.
  • It must be possible to get information used to efficiently search and navigate accross data from from different users. Both triplet and fulltext based searches are required.

XHTML Rendering in Vodyanoi

There are multiple posts on this topic in e-mentality blog. Mainly check these that are related to content based navigation. In short, process of rendering and its result can be described as follows.

  • Annotations of concepts will be processed in Wiki manner in order to link to other concepts/notebooks that are known to the Vodyanoi server.
  • Every link in the rendered page will contain not only URL but also URN of the target resource and (RDF) predicate that this link represents. For this purpose will be used e.g. rel attribute and/or rdf:resource & rdf:property attributes.
  • Notebook/concept will be annotated with DublinCore that will be associated with the rendered page using link element and its rel attribute.
  • A9/Wikipedia/FOAF project sub-ontologies will be used to link and annotate important keywords and abbreviations either using links or title attributes.
  • Pages should be rendered to XHTML 1.0 and CSS 2.0 (up to the extend that is reasonable and supported by browsers).
  • Body element will be identified with the ID attribute holding host information of the Vodyanoi server installation. It will enable clients to use custom CSS (e.g. for another media types).
  • Pages must be built with respect to disabled users e.g. through the code will be used label attributes enabling usage of speech engines.

As soon as there will be some time & I will have all relevant documents/specs perused, I plan to attach example XHTML page in here.

Annotation & Off-line

One of the features that I'm missing on (T)Wiki is off-line. In other words multiple users cann't work on a topic when they have no network connection to (T)Wiki server. I would like to distribute MindRaider data in P2P manner and enable off-line work and annotation of mind maps. To achieve this, the following approach and best practises will be used:

  • Only author of the notebook is allowed to create concepts in that notebook.
  • Anybody can annotate arbitrary concept of another author.
  • Only author of the concept can modify and refactor that concept.
  • Annotations are tightly bound to parent concept and parent annotations (strong relationship). This relationships can not be broken. Only concept to concept refactoring is allowed (weak relationship).
  • Neither concepts nor annotations can not be deleted. Although they can be marked as obsolete/discarded and moved under the thrash concept.

Summary

In other words, there is always somebody like notebook moderator (author) who publishes her/his notebook and who is looking for feedback. Other users may annotate his thoughts and provide various suggestions, but they can't change his work. It's up to the owner whether particular annotation will be incorporated into the concept or no.
